1. The "Who Am I?" Workaround (No Passwords Required)Since you don't have traditional user authentication, solve attribution using Device LocalStorage.How it works: The first time someone opens the app on their phone, show a quick 2-second modal: "Who's using this device?" with a grid of player names/avatars.The Logic: Store the chosen playerId in localStorage.The Result: Whenever that phone reacts to a log, uploads a photo, or spins a wheel, the frontend automatically attaches playerId to the request payload. You get individual player attribution without making anyone type a password.Note: Add a small avatar/name icon in the app header so anyone can tap it and switch identity if they swap phones.2. Photo Upload Architecture (Avatars & Photo Proof)Since you're hosting locally and serving over Tailscale Funnel, you don't need AWS S3 or external blob storage. You can save files directly to the mini PC's storage.The Backend Strategy (Node.js + Express)Storage: Use multer middleware on your Express server to save uploaded images into a local /uploads directory on your mini PC.Serving Files: Use express.static('uploads') so images are accessible via URL paths like [https://your-tailscale-funnel.ts.net/uploads/avatars/player123.jpg](https://your-tailscale-funnel.ts.net/uploads/avatars/player123.jpg).Database References: Instead of storing raw image binaries in SQL, store the relative file path string (e.g., /uploads/proof/event_45.jpg) in your DB tables.Database Schema AdjustmentsSQL-- Add avatar column to players
ALTER TABLE players ADD COLUMN avatar_url TEXT;

-- Add photo column to history/scores log
ALTER TABLE history_logs ADD COLUMN photo_url TEXT;
Critical Client-Side Tip (Frontend Compression)Modern phone cameras take 5MB–15MB photos. Uploading multiple full-size photos over a Tailscale Funnel can create slight delays.Solution: Use a lightweight JS library like browser-image-compression on the React frontend before sending the file in FormData. Compress images down to ~300KB/1080p width on the phone before hitting your Node backend.3.
